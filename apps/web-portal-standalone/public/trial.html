What to verify before go-live

What to verify before go-live

01 Owner and tenant routing

Confirm global users land in the owner console and tenant-scoped admins land in the tenant console without overlap.

02 Bot, worker, watcher, and optional agents

Validate runtime health, service readiness, and alert visibility from the owner console before tenant onboarding.

03 Catalog, delivery, and wallet workflows

Run tenant-scoped config, queue, wallet, and delivery recovery flows without touching owner-only controls.

04 Wallet, purchase, redeem, and Steam link

Walk through the player portal on mobile and desktop to confirm a clean, understandable transaction experience.

Before opening tenants

Before opening tenants

  1. Runtime health looks stable and major owner alerts are understood.
  2. Security, audit, and request anomaly pages are readable from one owner surface.
  3. Backup, restore preview, and guarded config changes still require deliberate action.
  4. Tenant fleet data and support case context can be exported without raw API digging.
Before opening the server community

Before opening the server community

  1. Tenant catalog, queue, dead-letter, and delivery support tools are usable without owner access.
  2. Player login, wallet, shop, redeem, and profile flows are understandable in both desktop and mobile layouts.
  3. Legacy workbenches are still reachable as fallback only, not the default operator route.
  4. Go-live messaging and restart communication are prepared before scale increases.
