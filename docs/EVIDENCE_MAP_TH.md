## Reading Rules

- if a claim is backed by code path + test + artifact, treat it as `verified`
- if a claim is backed by code path only, treat it as `implemented`
- if a claim depends on SCUM client state, Windows session state, or external infrastructure, treat it as `runtime-dependent`
- if there is no code, test, or artifact yet, treat it as `planned`
