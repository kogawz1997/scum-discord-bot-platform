Compatibility

What stays intact

  • Owner, tenant, player, and fallback paths keep their route behavior and runtime boundaries.
  • Request and response contracts, sessions, Discord SSO, TOTP, and runtime boundaries stay intact.
  • Critical ids, names, and JS bindings stay in place so the redesign changes structure without breaking behavior.
  • High-density emergency workbenches remain reachable as fallback, but no longer define the main experience.
