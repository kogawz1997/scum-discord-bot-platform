🧨 MASTER LIST: สิ่งที่ต้องแก้ทั้งหมด (ก่อนเรียก product-ready)

🔴 PHASE A — Core Critical (ต้องทำก่อน ไม่งั้นระบบพังง่าย)

นี่คือ 5 ตัวที่ “ห้ามข้าม” เด็ดขาด

1) 🔥 ลด dependency ของ console-agent

ปัญหา:
•พึ่ง SCUM client
•พึ่ง window
•พึ่ง Windows session

สิ่งที่ต้องทำ:
•agent liveness check + auto-restart
•detect window focus / resolution fail
•classify error (client crash vs command fail)
•queue fallback / quarantine
•retry strategy ที่ฉลาดขึ้น

👉 นี่คือ คอขวดอันดับ 1 ของทั้งระบบ

⸻

2) 🔥 ทำ tenant isolation ให้ “ครบทั้งระบบ”

ตอนนี้ยังมี shared path อยู่

สิ่งที่ต้องทำ:
•migrate ทุก tenant-aware module
•eliminate cross-tenant leakage risk
•enforce tenant context ทุก query
•test isolation จริง (ยิงหลาย tenant พร้อมกัน)

👉 ถ้ายังไม่จบ = ยังไม่ใช่ multi-tenant product-ready
⸻

3) 🔥 ขยาย native proof → cross-environment

ตอนนี้ proof ดี “ในเครื่องคุณ”

สิ่งที่ต้องทำ:
•test 2nd machine / 2nd server
•test หลาย config (latency / region)
•เพิ่ม item coverage
•verify database-per-tenant scenario

👉 ของจริงพังตรงนี้บ่อยสุด

⸻

4) 🔥 ทำ restore / rollback ให้ production จริง
ตอนนี้ยัง semi-manual

สิ่งที่ต้องทำ:
•snapshot restore test•rollback drill
•automated recovery flow
•disaster playbook

👉 เวลาระบบพัง = ไม่มีใครอยากเปิด README มานั่งอ่าน

⸻

5) 🔥 ทำ config control ให้ centralized (ผ่าน admin)

ตอนนี้ยังต้องแก้ .environment
สิ่งที่ต้องทำ:
•expose config ผ่าน admin panel
•validation + safe update
•audit config change
•hot reload support

👉 product จริงต้อง “คลิกแล้วจบ”

⸻

🟠 PHASE B — Production Hardening (ทำแล้วระบบจะนิ่งขึ้นเยอะ)
