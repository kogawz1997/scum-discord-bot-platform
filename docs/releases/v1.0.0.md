# Release Notes v1.0.0

Reference date: **2026-03-15**

## Summary

This release established the current split-runtime layout and the operational baseline used by the repository today.

## Main Changes

### Runtime

- Separated `bot`, `worker`, `watcher`, `admin web`, `player portal`, and `console-agent`
- Added health endpoints, topology checks, and smoke checks

### Admin and security

- Added DB-backed admin login
- Added Discord SSO, TOTP 2FA, and step-up auth
- Added session revoke and security event logging

### Persistence

- Added provider-aware Prisma generation and migration commands
- Added SQLite-to-PostgreSQL cutover tooling
- Moved the active workstation runtime to PostgreSQL

### Operations

- Added CI-backed verification artifacts
- Added restore preview and restore guardrails
- Added stronger runtime and audit visibility in admin

### Operator follow-up kept in the current repository state

- Added machine-local runtime install helpers for `Server Bot` and `Delivery Agent`
- Added runtime inventory reporting from the control-plane registry
- Added clearer two-machine runtime operator guidance for Machine A / Machine B installs

## Operator Impact

- Runtime responsibilities are clearer and easier to debug
- Verification status should now be taken from CI artifacts rather than handwritten counts in docs
- PostgreSQL becomes the expected runtime path for the active workstation

## Known Limits In This Release

- `agent` execution still depends on Windows session state and a live SCUM client
- Admin web still does not cover every env/config setting
- Tenant isolation was still application-level at this release point
- Visual evidence in the repository is still limited

## Deployment Conditions

- `NODE_ENV=production`
- `PERSIST_REQUIRE_DB=true`
- `PERSIST_LEGACY_SNAPSHOTS=false`
- split-origin deployment for admin and player surfaces
- successful `doctor`, `security:check`, and `readiness:prod`
