แก้ยอดเหรียญ จัดการสินค้า และปรับสถานะคำสั่งซื้อ

ตั้งจำนวนเหรียญ

กำหนดยอดใหม่แทนที่ยอดเดิมทันที

เพิ่มเหรียญ

ใช้แจกเหรียญกิจกรรมหรือชดเชยผู้เล่น

หักเหรียญ

หักยอดกระเป๋าโดยตรง (ตรวจสอบรหัสผู้ใช้ให้ถูกก่อนทุกครั้ง)

เพิ่มสินค้าใหม่

เลือกว่าสินค้าเป็น VIP หรือไอเทมในเกม ถ้าเป็นไอเทมสามารถเลือกจากรายการพร้อมไอคอนได้ทันที

ปรับราคาสินค้า

ใส่รหัสหรือชื่อสินค้าอย่างใดอย่างหนึ่ง

อัปเดตสถานะคำสั่งซื้อ

รองรับค่า รอดำเนินการ, ส่งแล้ว, คืนเงินแล้ว หรือสถานะที่คุณกำหนดเอง

ตั๋ว กิจกรรม และค่าหัว

เครื่องมือสำหรับทีมแอดมินในการจัดการคอมมูนิตี้

รับเรื่องทิคเก็ต

ระบุรหัสห้องทิคเก็ตและรหัสผู้รับเคส

ปิดทิคเก็ต

ปิดทิคเก็ตจากหน้าเว็บโดยตรง (จะแจ้งข้อความในห้องถ้าส่งได้)

สร้างกิจกรรม

ประกาศกิจกรรมพร้อมเวลาและรางวัล

เริ่มกิจกรรม

ใส่รหัสกิจกรรมที่ต้องการเปิด

จบกิจกรรม

ใส่รหัสกิจกรรมเพื่อปิดสถานะกิจกรรม

เพิ่มผู้เข้าร่วมแบบกำหนดเอง

ใช้กรณีผู้เล่นเข้าร่วมแต่ระบบไม่บันทึกอัตโนมัติ

สร้างค่าหัว

ระบบจะเพิ่มรายการค่าหัวใหม่ทันที

ยกเลิกค่าหัว

ใส่รหัสค่าหัวที่ต้องการยกเลิก

ผู้เล่น VIP และระบบแลกรางวัล

จัดการการลิงก์ Steam, การเป็นสมาชิก VIP และโค้ดแลกรางวัล

ลิงก์ Steam กับ Discord

ใช้ SteamID64 จริงเพื่อป้องกันข้อมูลชนกัน

ลบลิงก์ Steam/Discord

ใส่ SteamID หรือ UserID อย่างน้อย 1 ค่า

ตั้งค่า VIP

เพิ่มสิทธิ์ VIP ให้ผู้เล่นตามจำนวนวัน

ถอด VIP

ลบสถานะ VIP ของผู้เล่นทันที

สร้าง/แก้โค้ดแลกรางวัล

ถ้าเลือกเหรียญให้กรอกจำนวน, ถ้าเลือกไอเท็มให้กรอกรหัสไอเท็ม

รีเซ็ตการใช้งานโค้ดแลกรางวัล

ทำให้โค้ดกลับมาใช้ได้ใหม่ตามเงื่อนไขระบบ

การดูแล, สถิติ และต้อนรับ

จัดการบันทึกลงโทษและปรับสถิติผู้เล่นด้วยมือ

เพิ่มบันทึกลงโทษ

ประเภทตัวอย่าง: เตือน, ปิดแชท, ไทม์เอาต์, แบน, หมายเหตุ

เพิ่มคิล

บวกสถิติคิลให้ผู้เล่นแบบกำหนดเอง

เพิ่มตาย

บวกสถิติตายให้ผู้เล่นแบบกำหนดเอง

เพิ่มเวลาเล่น

หน่วยเป็นนาที เช่น 60 = 1 ชั่วโมง

ยกเลิกสิทธิ์รับแพ็กต้อนรับรายคน

ให้ผู้เล่นคนนี้กลับมารับแพ็กต้อนรับได้อีกครั้ง

อัปเดตสถานะเซิร์ฟเวอร์ (กำหนดเอง)

กรอกเฉพาะช่องที่ต้องการแก้ได้

ส่งของ RCON และระบบเช่ามอไซ

จัดการคิวส่งของอัตโนมัติและสั่งรีเซ็ตระบบเช่ารถ

สถานะ runtime ส่งของ

ตรวจ queue, dead-letter, worker และสุขภาพของ agent/RCON แบบสรุป

ยังไม่มีข้อมูล runtime ส่งของ

พรีวิวคำสั่งส่งของ

เช็กว่า item หนึ่งชิ้นจะถูกแปลงเป็นคำสั่งอะไร ก่อนกดซื้อจริง

ยังไม่มีผลพรีวิวคำสั่ง

ส่ง Test Item เข้าตัวแอดมิน

ใช้ตรวจ execution chain แบบสด ใน agent mode ของจะเสกใกล้ตัวแอดมินที่ล็อกอินค้างอยู่

ยังไม่มีผล test send

Dead-Letter ล่าสุด

กด retry / delete / ดู log ได้จากแถวโดยตรง

Command Log / Purchase Code

เลือก purchase code จากคิวหรือ dead-letter เพื่อดูสถานะ, history และ command log ที่ยิงจริง

ยังไม่ได้เลือก purchase code

เพิ่มคิวส่งของ

เพิ่มคำสั่งซื้อเข้าคิวส่งของอีกครั้ง

ลองส่งของใหม่ทันที

สั่งให้งานนั้นลองส่งใหม่ทันที

Retry จาก Dead-Letter

ลบรายการ Dead-Letter

ยกเลิกงานในคิว

Metrics Timeline
